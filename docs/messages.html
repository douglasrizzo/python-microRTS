The unit type table stores the unit types the game can have. It also determines the attributes of each unit type. The unit type table determines the balance of the game.

Action¶

Every time the pyrts.server.Server.get_action() method of the user-created controller is called, a list of actions must be returned, one action for each unit in the game. The list is a standard Python list and each action is a Python dict objects containing the unit ID and another dict containing the action and its parameters. Action names and codes are listed in pyrts.server.Action. Action parameters, which are mostly directions to apply the actions, are listed in pyrts.server.Direction.

[
    {
        "unitID": 0 # the unit that you wish to control
        "unitAction": {
            'type': 1 # the type of action (this is defined in the unit type table)
            'parameters': 0 # the parameters for the action (this is defined in the unit type table)
        }
    },
    {
        # more actions for different unit IDs
    }
]
